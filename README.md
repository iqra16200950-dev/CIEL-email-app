# CIEL-email-app
Hey and WELCOME TO CIEL!

I wanted and email app that didnt feel excessive and loud.
Everything rn is super cluttered and heavy and i wanted smething that felt like opening your windows and seeing a pastel sky .And thta when ciel (which means sky in french)came into existence .I also preferred the idea of messaging through username instead of LONGG email addresses,this would be a lil more personal.I created this specifically keeping in mind my long distance friends ,who im in contact with but it isnt really as fun.Me and my best friend wrote letters to each other even when we met face to face together in school,but ever since i had to leave and we couldnt do that , i wanted this app to recreate the feeling again but in an online based way.

*Design*
The whole app is based on soft pastel sky vibes ,nothing harsh ,nothing loud.
all the assets are drawn by me , which made it feel a lot mor personal but also took wayy longer than expected .

*Features*
>Sign up/Login
>Send messages using usernames
>Inbox(recieved messages)
>View sent messages
>Save drafts
>Trash messages and delete them permenantly in the trash section.

*How I built it*
This was supposed to be an electron desktop app using electron and i even wrote  the code for it .But, halfway through,i realised i was sorta overcomplicating it and honestly just getting tired .The last time i used elctron i was really overwhelmed as i had a lot of issues(i will try but not for this lol) , i wanted this poject to be calming ,so i scaled it down and built it as a lightweight web app,mostly using a single html file with simple logic .It ended up being more manageable and actually got finished, which mattered to me more.

*What i struggled with*
Keeping the aesthic minimal without making it feel empty,
structuring the message system,
stayng consistent with design style,
and also debugging for hours only to find out it was just a typo that was the issue.

*What I learnt*
>Simple projects still require a lot of thought
>Cutting down your idea is sometimes better
>Building or drawing your own assets makes a big difference in how the project feels
>Using a single file might be better for certain projects.
>i also learnt how to debug and implement new systems like ogin,sign up and all that stuff.

Ciel isnt meant to replace big email platforma,its just a small project that explore what email could feel like if it was quiter,softer and a bit more personal.
